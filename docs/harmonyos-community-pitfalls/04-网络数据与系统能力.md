# 网络、数据与系统能力踩坑

## 1. HTTP / 明文与配置

- **Stage 与 FA 模型差异**：历史帖子提到早期 **Stage** 文档相对少、**HTTP 明文** 等配置在 **FA** 示例更多；新版本请以当前 `module.json5` / 网络安全配置为准。
- **文件上传**：社区指出部分官方 Demo 中 **Header** 类型与运行时要求不一致（如应为对象却写成 string），需对照 **最新示例** 或自行修正。

*来源：IT 营《ArkTs api8 踩坑记录》等。*

## 5. 键盘、输入框与列表（API 12+）

- **键盘弹起** 时，底部输入框与上方 **List** 无法同步滚动/避让：社区尝试方向包括 **`adjustPan`** 类窗口/输入法模式、**键盘高度监听** 后手动 `scroll`、**expandSafeArea** 与安全区（需结合 `windowStage` 与页面结构）。
- **bindSheet / UIExtension + TextArea**：长文本滚动时可能带动 **半屏 sheet** 一起滚；讨论方案包括 **内层嵌套 Scroll**、减少 UIExtension 组合方式等（以具体版本行为为准）。

*来源：CSDN 问答、DEV Community 日常问答汇总等。*

## 6. 分布式与多设备（概述）

- **设备发现不到**：局域网、防火墙、蓝牙/分布式开关；社区建议同网段、关干扰防火墙规则。
- **数据同步冲突**：多设备同时写需 **冲突解决**、**事务** 或业务层合并策略。
- **调试**：分布式断点难，可配合 **日志** 与官方 **分布式调试** 能力。

*来源：腾讯云转载《从 ArkTS 到分布式调试》等。*

## 7. 从 Android 迁移时的网络与并发（英文博客归纳）

- **`@ohos.net.http`**：长连接/请求需在 **`finally` 等路径调用 `destroy()`** 释放底层资源，否则长期运行可能 **泄漏**（社区/英文文章强调，需以官方示例为准）。
- **并发模型**：与 Kotlin 协程共享可变内存不同，ArkTS 侧多使用 **TaskPool / Worker**，跨线程传递需 **序列化消息**，不能把带状态的同一对象直接传入 Worker。

*来源：英文迁移指南类博客（仅作架构提醒，实现以华为文档为准）。*
