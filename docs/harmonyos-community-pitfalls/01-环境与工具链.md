# 环境与工具链踩坑（DevEco / 模拟器 / ohpm / 签名）

## 1. DevEco Studio 与构建

- **文档与 Hvigor 常见问题**：官方提供 IDE 与 Hvigor 的 FAQ，遇到同步失败、插件问题可优先查文档中心「编译构建」相关条目。
- **Release 与 Debug 混淆**：断点进不去、调试行为异常时，社区常见原因是误用 **Release** 构建或签名与调试模式不匹配，需切回 **Debug** 并确认运行配置。

## 2. 签名与真机调试（含错误码 10106002）

社区多次讨论 **Error Code: 10106002** 类调试部署失败，常见线索包括：

- 删除旧签名后勾选 **Automatically generate signature** 重新生成调试证书。
- 检查项目级 `build-profile.json5` 中 `signingConfigs` 是否多套冲突；可尝试清空后仅保留自动签名再试。
- **错用 Release 包做调试**：日志若出现与 `non-app-provision`、Release 相关提示，需卸载旧应用，改用 **Debug 证书**或自动签名的 Debug 产物再装。
- **设备未授权调试**：USB 调试开启后，在 DevEco **设备管理**中对设备执行授权；必要时 `adb kill-server` / `adb start-server` 后重试。
- 核对 `module.json5` 中 `compileSdkVersion` / `compatibleSdkVersion` 与真机系统是否匹配。

*来源线索：IT 营论坛等帖子中开发者与回复归纳。*

## 3. ohpm 安装失败（ETIMEDOUT、00617101、ENOENT 等）

| 现象 | 常见对策（社区经验） |
|------|----------------------|
| `connect ETIMEDOUT` | 检查代理/防火墙；切换 registry，例如 `https://ohpm.openharmony.cn/ohpm/`；DevEco 内 **Settings → Build → Ohpm → Optimize config** 配置镜像与代理。 |
| `00617101` / Fetch 失败 | 确认包名与版本在仓库存在；`ohpm config get registry`；清理缓存后重装。 |
| `ohpm` 未识别 / 安装脚本失败 | 确认 ohpm 在 PATH 或 IDE 指定目录；Windows 下曾有与 **cmd 自动 chcp 65001** 注册表项冲突的讨论，可排查 `Command Processor` 的 `AutoRun`。 |
| SSL / 证书问题 | 社区有在 `~/.ohpm/.ohpmrc` 中将 **strict_ssl** 设为 `false` 的做法（仅建议在理解中间人风险后用于内网/调试）。 |

通用排查：`ohpm cache clean` 或 `ohpm cache clean --force`；`ohpm install --loglevel verbose` 看 404/超时/拒绝连接。

*来源线索：CSDN 问答、IT 营论坛、博客园《ohpm 安装失败》等。*

## 4. 模拟器、预览器与生命周期

- **模拟器无法访问网页**：在 `module.json5` 中声明 **`ohos.permission.INTERNET`**（即使属于基础权限，也需显式声明），否则联网相关能力异常。
- **Previewer 白屏/崩溃**：若仅在预览器出问题，检查是否在 `onWindowStageCreate` 等处依赖 **`AppStorage` 初始化**；预览器可能不走完整 Ability 生命周期，需 **空值保护** 或避免在预览路径执行完整初始化。
- **模拟器与真机差异**：布局、性能、传感器等在真机上需复测；社区强调 **vp** 等单位与真机分辨率差异。

## 5. HSP 模块与依赖

- **HSP 无法引用**：主工程 `oh-package.json5` 中通过 **`file:./library`** 等形式引入子模块后执行 **`ohpm install`**；路径与包名需一致。

## 6. 其他环境类问题（论坛零散反馈）

- **调测助手安装失败**（如内部错误码）：以华为开发者论坛对应帖子的官方/社区回复为准，通常涉及环境、驱动或工具版本。
- **签名/Profile 与 bundleName 不一致**：上架或安装失败时核对应用 ID、Profile 与工程 `bundleName`。
